About

The Fortnite-Stats Bot API is an unofficial Fortnite API. It can be used for retrieving in-game stats of a player.
The API is actually maintained and owned by @y3n

Getting started

Authentication

All the APIs requires one mandatory header:
X-Key - the header that contains your API Key.

You can generate your own API Key by simply sending !getapikey in private message to the Bot.
Basic API access is limited to 2000 calls/day and to 1 req per second.

You can check your daily quota left with the X-DayQuotaLeft header sent by the server on each response.

Your API Key is unique and you can get only one key, so don't give it!

Response

Responses will be in the JSON format. See Basic example for an example of response.

Basic example

This example shows how to retrieve stats from a player using GET /v2/player/{playerName} endpoint.

  • Node.js
  • PHP
  • Ruby
  • Java
  • Python 
var request = require('request');

var playerName = "robi62"; // Player in-game name
var apiKey = "Your API Key"

var options = {
    method: "GET",
    url: `https://fortnite.y3n.co/v2/player/${playerName}`,
    headers: {
        'User-Agent': 'NodeJS Request',
        'X-Key': apiKey
    }
}

request(options, (error, response, body) => {
    if (!error && response.statusCode == 200) {
        var stats = JSON.parse(body);
        console.log(stats);
    } else {
        console.log(error);
    }
})

HTTP Codes

The API uses HTTP status codes to indicate success or failure of an API call. In general, status codes in the 2xx range means success. Commonly used HTTP status codes are listed below.

Status Code Description
200 Success
The request was successfully completed.
503 Too many requests at the same time.
The API is currently limited to 1 req/sec. Be a little bit more slow!
429 Daily quota exceeded.
The actual quota is 2000 calls/day. Contact y3n to upgrade.
403 Forbidden access.
The API Key is missing in the headers or incorrect API Key.
404 User not found.
Verify that the user is using his Epic Account name and not PSN/Xbox account name.

Endpoints

Lookup player stats

GET
/v2/player/{playerName}
Description Shows player profile and stats
Parameters {playerName} - Player name to lookup
Returned values
  • profile - Player profile informations
    UNAVAILABLE
  • stats - Player stats
    • pc|ps4|xbox - Stats on different platforms
      • solo - Solo mode stats
      • duo - Duo mode stats
      • squad - Squad mode stats
      • all - All modes combined stats

      • solo|duo|squad|all values:
        • kills - Number of kills
        • matchesPlayed - Total of played matches
        • lastMatch - DateTime of the last match played
        • minutesPlayed - Total of minutes played
        • wins - Total of winned matches
        • top10 - Total of Top 10
        • top25 - Total of Top 25
        • deaths - Total of deaths
        • kpd - Kills Per Death
        • kpm - Kills Per Minute
        • tpm - Time per Match
        • score - Player score
        • winRate - Wins ratio
  • stw - Save The World stats
    UNAVAILABLE
  • lastUpdate - Last update of the stats
  • userID - Player ID
  • displayName | displayNameLowerCase - Player full name
  • lastLookup | lastLookupUpdate - DateTime of the last lookup of player stats
  • discordID - Player Discord ID (if it's linked)

Lookup player matches

GET
/v2/player/{userID}/matches
Description Shows player matches
Parameters {userID} - User ID to lookup
Returned values
  • _id - Match ID
  • displayName | displayNameLowerCase - Player full name
  • userID - Player ID
  • device - Match platform (pc|ps4|xbox)
  • modes - Match modes
    • solo|duo|squad - Number of matches played in the specified mode
  • matchesPlayed - Number of matches played
  • kills - Player kills
  • score - Player score
  • wins - Player wins
  • minutesPlayed - Player minutes played
  • tops - Player top
    • top10|top25 - Number of tops in all matches
  • deaths - Number of player deaths

/v2/shop
Description Returns all shop items
Parameters No parameters
Returned values
  • refreshIntervalHrs - Interval of refreshing

      • devName - Dev name

        • currencySubType - Currency sub type
        • regularPrice - Item regular price
        • finalPrice - Item final price
        • saleExpiration - DateTime of Item sale expiration
        • basePrice - Item base price
      • categories - Item categories
      • dailyLimit - Item daily limit (-1 for null)
      • weeklyLimit - Item weekly limit (-1 for null)
      • monthlyLimit - Item monthly limit (-1 for null)
      • appStoreId - Item app store ID
      • requirements - Item requirements
      • metaInfo - Item meta info
        • key - Key
        • value - Value
      • catalogGroup - Item catalog group
      • catalogGroupPriority - Item catalog group priority
      • sortPriority - Item sort priority
      • title - Item title
      • shortDescription - Item short description
      • description - Item description
      • displayAssetPath - Item description
      • itemGrants - Item grants
  • br - Battle Royal items
    • daily - Daily BR items
      • name - Item name
      • imgURL - Item image URL
      • imgPNGURL - Item image URL in PNG
      • subname - Item subname
      • price - Item price
      • rarity - Item rarity
      • cosmeticType - Item cosmetic type

Getting Fortnite status

GET
/v2/gamestatus
Description Shows actual Fortnite status
Parameters No parameters
Returned values
  • status - Actual status of Fortnite (UP|DOWN)
  • message - Message about the status
  • maintenanceUri - Maintenance URI
Example request X-Key: DGHae1kl26sj5sd6dfim
GET /v2/gamestatus
Example response 
{  
   "status":"UP",
   "message":"Fortnite is up.",
   "maintenanceUri":null
}
